CSS Specificity

คอนเซปต์การเลือก CSS Selector ที่มี “ค่าพลัง” เยอะที่สุด มาใช้กับ Element บนหน้าเว็บไซต์

ภาษา CSS อ่านโค้ดจาก บน – ล่าง ดังนั้นถ้าเขียน CSS Selector ซ้ำกัน อันล่างจะถูกนำไปใช้งานเพราะใหม่กว่า

แต่ความพิเศษของ CSS คือ Selector ที่แตกต่างกัน แต่ละ Selector จะมี “ค่าพลัง” ไม่เหมือนกัน และเมื่อไหร่ที่ “ค่าพลัง” ต่างกัน ลำดับการเขียนโค้ดจะไม่มีผลทันที

การคำนวณ "ค่าพลัง"

ค่าพลังจะคำนวณจากการใช้ class, ID, !important หรือการซ้อน Element หลาย ๆ ชั้น

    1. เขียน CSS ใส่ใน style : +1000 point

    2. ใส่ 1 ID : +100 point

    3. ใส่ 1 class : +10 point

    4. ใส่ 1 Element (Tag HTML) : +1 point

ยิ่ง Selector นั้นเจาะจงเท่าไหร่ ยิ่งมีค่าพลังเยอะเท่านั้น


ตัวอย่าง

< div class="box" id="redbox">

.box { background: white; }

#redbox { background: red; }

จะได้กล่องสีแดงมา เนื่องจากจาก id มีค่าพลังมากกว่า class

การเขียน CSS ไม่ให้ตีกับ CSS Specificity

อย่าใช้ #ID ใน Selector

การใช้ ID ซึ่งเพิ่มค่าพลังถึง 100 คะแนน จะทำให้ความแตกต่างระหว่าง Selector มากเกินไป และเขียนทับได้ยาก

ถ้าจะทับต้องเขียน ID, Inline CSS, หรือ !important เท่านั้น ซึ่งทำให้โค้ดหนักเข้าไปใหญ่

อย่าเขียน Selector ซ้อนกันโดยไม่จำเป็น

เขียน .box-inside { } จะดีกว่าเขียน .box .box-inside { }

ซึ่งเป็นการซ้อนโดยไม่จำเป็น และทำให้ค่าพลังสูงกว่าเดิมโดยไม่จำเป็น ยกเว้นว่าแบบแรกไม่ทำงาน

อย่าเขียน Selector ชั้นเดียวแบบเจาะจงเกินไป

เขียน .box { } จะดีกว่าเขียน div.box { }

ซึ่งเป็นการเจาะจงเกินไป ทำให้ค่าพลังสูงกว่าเดิมโดยไม่จำเป็น ยกเว้นว่าแบบแรกไม่ทำงาน

ใช้ Class ให้บ่อยเข้าไว้

Class เป็นการเขียน Selector ที่ค่าพลังไม่สูงเกินไป จะเขียนทับก็ไม่ยาก หรือจะสามารถนำมาใช้ใหม่ก็ทำได้ง่าย
